# WePlus 部署指南

> 本文档整合了 WePlus 项目的完整部署方案，涵盖本地开发部署、生产环境部署、Railway 云部署和部署检查清单。
> 合并来源：`DEPLOYMENT.md` + `frontend2/部署指南.md` + `deploy/RAILWAY_部署指南.md` + `deploy/DEPLOYMENT_CHECKLIST.md`

## 第一章：部署概览与环境准备

### 1.1 部署架构

WePlus 支持多种部署方式：

| 部署方式 | 适用场景 | 复杂度 |
|---------|---------|--------|
| 本地开发 | 开发调试 | 低 |
| Docker Compose | 生产环境 | 中 |
| Railway 云部署 | 快速上线 | 低 |
| 阿里云 ECS | 生产环境 | 高 |

### 1.2 环境要求

- Python 3.11+
- Node.js 18+
- PostgreSQL 14+ (含 pgvector 扩展)
- Docker 20.10+ (可选)
- NPM 9+ (可选)

### 1.3 环境变量配置

在项目根目录创建 `.env` 文件（参考 `.env.example`）：

```env
# 应用配置
SECRET_KEY=your-secret-key-here
ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000
DEBUG=true

# 数据库配置
DATABASE_URL=postgresql://user:password@localhost:5432/weplus
DATABASE_POOL_SIZE=10
DATABASE_MAX_OVERFLOW=20

# AI 服务配置
OPENAI_API_KEY=your-openai-api-key
OPENAI_MODEL=gpt-3.5-turbo
EMBEDDING_MODEL=text-embedding-3-small

# JWT 配置
JWT_SECRET_KEY=your-jwt-secret
JWT_ALGORITHM=HS256
JWT_EXPIRATION_MINUTES=30

# 管理员配置
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=your-admin-password

# 可选配置
SENTRY_DSN=your-sentry-dsn  # 错误监控
REDIS_URL=redis://localhost:6379/0  # 缓存
SMTP_SERVER=smtp.example.com  # 邮件
SMTP_PORT=587
SMTP_USER=your-email
SMTP_PASSWORD=your-email-password
```

## 第二章：本地开发部署

### 2.1 后端部署

#### 步骤1: 克隆项目
```bash
git clone <repository-url>
cd weplus-12.0/backend
```

#### 步骤2: 创建虚拟环境
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/Mac
python3 -m venv venv
source venv/bin/activate
```

#### 步骤3: 安装依赖
```bash
pip install -r requirements.txt
```

#### 步骤4: 配置环境变量
```bash
cp .env.example .env
# 编辑 .env 文件，填入实际的配置值
```

#### 步骤5: 初始化数据库
```bash
# 确保 PostgreSQL 已启动并创建了数据库
python init_db.py

# 或使用 Alembic 迁移
alembic upgrade head
```

#### 步骤6: 启动服务器
```bash
# 开发模式 (热重载)
uvicorn main:app --reload --host 0.0.0.0 --port 8000

# 生产模式
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

### 2.2 前端部署

#### 步骤1: 安装依赖
```bash
cd frontend2
npm install
```

#### 步骤2: 配置代理
前端通过 Vite 代理将 API 请求转发到后端。在 `vite.config.ts` 中配置：

```typescript
export default defineConfig({
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
    },
  },
});
```

#### 步骤3: 启动开发服务器
```bash
npm run dev
```

#### 步骤4: 构建生产版本
```bash
npm run build
# 构建产物在 dist/ 目录
```

### 2.3 验证部署

访问以下地址确认部署成功：
- 前端应用: http://localhost:5173
- 后端 API: http://localhost:8000
- API 文档: http://localhost:8000/docs
- 健康检查: http://localhost:8000/api/healthz

## 第三章：生产环境部署

### 3.1 Docker 部署

#### 使用 Docker Compose (推荐)

```bash
# 从项目根目录运行
docker-compose -f deploy/docker-compose.yml up -d

# 查看日志
docker-compose -f deploy/docker-compose.yml logs -f

# 停止服务
docker-compose -f deploy/docker-compose.yml down
```

#### 单容器部署

```bash
# 构建镜像
docker build -t weplus:latest .

# 运行容器
docker run -d \
  --name weplus \
  -p 80:80 \
  -e DATABASE_URL=postgresql://user:pass@host:5432/weplus \
  -e SECRET_KEY=your-secret-key \
  -e ALLOWED_ORIGINS=https://yourdomain.com \
  -e OPENAI_API_KEY=your-openai-api-key \
  weplus:latest
```

#### Docker Compose 配置参考

```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "80:80"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/weplus
      - SECRET_KEY=${SECRET_KEY}
      - ALLOWED_ORIGINS=${ALLOWED_ORIGINS}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
    depends_on:
      - db
    volumes:
      - uploads:/app/backend/uploads
      - logs:/app/backend/logs

  db:
    image: postgres:15
    environment:
      - POSTGRES_DB=weplus
      - POSTGRES_USER=weplus
      - POSTGRES_PASSWORD=${DB_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./deploy/init.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "5432:5432"

volumes:
  postgres_data:
  uploads:
  logs:
```

### 3.2 Nginx 配置

#### 单容器 Nginx 配置 (部署用)

```nginx
server {
    listen 80;
    server_name _;

    # 前端静态文件
    root /app/frontend2/dist;
    index index.html;

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # SSE 支持 (用于 RAG 流式响应)
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 86400s;
    }

    # WebSocket 支持
    location /ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 静态资源缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 3.3 Supervisord 配置 (单容器进程管理)

```ini
[supervisord]
nodaemon=true
user=root
logfile=/var/log/supervisor/supervisord.log
pidfile=/var/run/supervisord.pid

[program:uvicorn]
command=uvicorn main:app --host 127.0.0.1 --port 8000 --workers 4
directory=/app/backend
user=root
autostart=true
autorestart=true
stderr_logfile=/var/log/uvicorn.err.log
stdout_logfile=/var/log/uvicorn.out.log
environment=PYTHONPATH="/app/backend:$PYTHONPATH"

[program:nginx]
command=nginx -g "daemon off;"
user=root
autostart=true
autorestart=true
stderr_logfile=/var/log/nginx.err.log
stdout_logfile=/var/log/nginx.out.log
```

## 第四章：Railway 云部署

### 4.1 部署概述

Railway 是一个现代化的云部署平台，支持从 GitHub 仓库自动部署。WePlus 针对 Railway 进行了优化，支持一键部署。

### 4.2 前提条件

1. 注册 Railway 账号 (https://railway.app)
2. 连接 GitHub 仓库
3. 准备 PostgreSQL 数据库 (Railway 提供)

### 4.3 部署步骤

#### 方法1: 一键部署 (推荐)

[![Deploy on Railway](https://railway.app/button.svg)](https://railway.app/template/your-template-id)

#### 方法2: 手动部署

1. **在 Railway 创建新项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择你的 WePlus 仓库

2. **添加 PostgreSQL 数据库**
   - 点击 "New" → "Database" → "PostgreSQL"
   - Railway 会自动创建数据库并提供连接字符串

3. **配置环境变量**
   ```
   DATABASE_URL=postgresql://...  # Railway 自动提供
   SECRET_KEY=你的安全密钥
   ALLOWED_ORIGINS=https://你的域名.railway.app
   OPENAI_API_KEY=你的OpenAI密钥
   JWT_SECRET_KEY=你的JWT密钥
   ```

4. **配置构建命令**
   ```bash
   # 后端
   cd backend && pip install -r requirements.txt
   
   # 前端
   cd frontend2 && npm install && npm run build
   ```

5. **配置启动命令**
   ```bash
   cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT
   ```

6. **部署完成**
   - Railway 会自动构建和部署
   - 部署完成后会生成 `.railway.app` 域名
   - 可以配置自定义域名

### 4.4 Railway 配置参考

**railway.json**:
```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "DOCKERFILE",
    "dockerfilePath": "Dockerfile"
  },
  "deploy": {
    "numReplicas": 1,
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10,
    "healthcheckPath": "/api/healthz",
    "healthcheckTimeout": 300
  }
}
```

### 4.5 部署后验证

```bash
# 运行部署后检查脚本
python deploy/checks/railway_post_deploy_check.py \
  --base-url https://你的项目.railway.app

# 手动验证
curl https://你的项目.railway.app/api/healthz
curl https://你的项目.railway.app/api/rag/health
```

### 4.6 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 部署失败 | 构建命令错误 | 检查 railway.json 配置 |
| 数据库连接失败 | DATABASE_URL 未正确设置 | 检查环境变量 |
| 静态文件 404 | 前端未正确构建 | 检查构建命令 |
| 健康检查失败 | 应用启动超时 | 增加 healthcheckTimeout |

## 第六章：运维指南

### 6.1 日志管理

```bash
# 查看应用日志
tail -f backend/logs/app.log

# 查看 Nginx 访问日志
tail -f /var/log/nginx/access.log

# 查看 Nginx 错误日志
tail -f /var/log/nginx/error.log

# 使用 journald 查看
journalctl -u weplus -f
```

### 6.2 备份与恢复

```bash
# 数据库备份
pg_dump -h localhost -U weplus_user -d weplus -F c -f backup_$(date +%Y%m%d).dump

# 数据库恢复
pg_restore -h localhost -U weplus_user -d weplus -c backup_20240101.dump

# 文件备份
tar -czf uploads_backup_$(date +%Y%m%d).tar.gz backend/uploads/
```

### 6.3 更新升级

```bash
# 拉取最新代码
git pull origin main

# 更新后端依赖
cd backend && pip install -r requirements.txt

# 更新前端依赖
cd frontend2 && npm install

# 执行数据库迁移
alembic upgrade head

# 重启服务
docker-compose restart
```
