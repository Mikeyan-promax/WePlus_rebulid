## 第一章：项目概述与设计目标

### 1.1 项目简介

WePlus 是一个基于 RAG (Retrieval-Augmented Generation) 技术的校园智能助手平台，旨在为校园用户提供智能问答、学习资源管理和文档处理等一站式服务。

### 1.2 核心目标

1. **智能化问答**: 基于上传文档的智能问答系统
2. **资源管理**: 多格式文档的上传、存储、检索和管理
3. **用户体验**: 简洁现代的界面设计，流畅的交互体验
4. **可扩展性**: 模块化设计，便于功能扩展和维护
5. **生产就绪**: 完善的监控、日志和部署方案

### 1.3 设计原则

- **前后端分离**: 独立开发、独立部署、独立扩展
- **API 驱动**: 统一 RESTful API 接口设计
- **分层架构**: 清晰的层次划分，关注点分离
- **类型安全**: TypeScript + Pydantic 双重验证
- **安全优先**: JWT 认证、权限控制、数据加密

## 第四章：架构设计决策

### 4.1 关键设计决策

#### 决策1: PostgreSQL + pgvector 统一数据存储

**背景**: 初始使用 SQLite + ChromaDB 双数据库方案

**决策**: 统一迁移到 PostgreSQL + pgvector

**理由**:
- 统一数据管理，简化架构
- ACID 事务支持，确保数据一致性
- 成熟的备份恢复方案
- 支持水平扩展

#### 决策2: FastAPI 异步框架

**背景**: 需要处理大量 I/O 操作 (文件上传、AI 调用、数据库查询)

**决策**: 使用 FastAPI 异步特性

**理由**:
- 原生异步支持，高性能
- 自动 API 文档生成 (Swagger/OpenAPI)
- Pydantic 模型验证
- 依赖注入系统

#### 决策3: JWT 无状态认证

**背景**: 需要支持前后端分离和 API 访问

**决策**: 采用 JWT Token 认证

**理由**:
- 无状态，易于扩展
- 支持跨域访问
- Token 可携带用户信息
- 支持过期时间设置

#### 决策4: RAG 流式响应 (SSE)

**背景**: AI 问答需要实时显示生成结果

**决策**: 使用 Server-Sent Events (SSE)

**理由**:
- 实时性：逐字显示 AI 生成内容
- 简单性：基于 HTTP 协议
- 兼容性：前端 EventSource API 原生支持
- 资源效率：不需要 WebSocket 的复杂连接管理

### 4.2 安全性设计

#### 认证与授权
- 密码使用 bcrypt 哈希存储
- JWT Token 包含用户角色信息
- API 路由通过依赖注入进行权限验证
- 敏感操作记录审计日志

#### 数据安全
- 所有 API 支持 HTTPS
- 数据库连接支持 SSL
- 用户密码绝不明文存储
- 文件上传类型和大小限制

#### API 安全
- CORS 白名单配置
- 请求频率限制
- 输入数据验证 (Pydantic)
- SQL 注入防护 (SQLAlchemy ORM)

### 4.3 性能优化策略

#### 数据库优化
- 连接池管理 (QueuePool)
- 慢查询优化 (EXPLAIN ANALYZE)
- 合理索引设计 (B-tree + IVFFlat/HNSW)
- 定期 VACUUM ANALYZE

#### 应用优化
- 异步 I/O 操作
- 响应数据压缩
- 静态资源缓存
- API 响应缓存 (Redis)

#### 前端优化
- 代码分割 (Lazy Loading)
- 图片懒加载
- 组件备忘录 (React.memo)
- 虚拟列表 (大数据量)

### 5.2 核心业务流程

#### 文档上传与处理流程
```
用户上传文档
    │
    ▼
文件类型验证 → 文件大小检查 → 保存到本地
    │
    ▼
文档解析 (PDF/Word/Excel/PPT/TXT)
    │
    ▼
文本分块 (Chunking)
    │
    ▼
生成嵌入向量 (OpenAI Embedding API)
    │
    ▼
存储到 pgvector (DocumentEmbeddings表)
    │
    ▼
更新文档状态为 "ready"
    │
    ▼
返回上传成功响应
```

#### RAG 问答流程
```
用户发送问题
    │
    ▼
生成问题嵌入向量
    │
    ▼
pgvector 相似度搜索 (Top-K)
    │
    ▼
获取相关文档片段
    │
    ▼
构建 Prompt (问题 + 上下文)
    │
    ▼
调用 OpenAI Chat API (流式响应)
    │
    ▼
逐字返回 SSE 响应
    │
    ▼
保存聊天历史
```

## 第六章：开发方案

### 6.1 开发环境搭建

```bash
# 后端
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# 编辑 .env 配置数据库和 API 密钥
uvicorn main:app --reload --host 0.0.0.0 --port 8000

# 前端
cd frontend2
npm install
npm run dev
```

### 6.2 开发流程

1. **需求分析**: 明确功能需求和技术方案
2. **接口设计**: 先定义 API 接口，生成文档
3. **后端实现**: 实现 API 路由和服务逻辑
4. **前端实现**: 实现 UI 组件和状态管理
5. **联调测试**: 前后端联合调试
6. **代码审查**: 代码审查和质量检查
7. **部署上线**: 构建和部署

### 6.3 代码规范

- **后端**: 遵循 PEP 8，使用 Black 格式化
- **前端**: ESLint + Prettier，TypeScript 严格模式
- **API**: RESTful 规范，统一响应格式
- **提交**: 约定式提交 (Conventional Commits)

### 6.4 测试策略

- **单元测试**: pytest (后端) / Vitest (前端)
- **集成测试**: API 端点测试
- **端到端测试**: 关键用户流程测试
- **性能测试**: 负载和压力测试
