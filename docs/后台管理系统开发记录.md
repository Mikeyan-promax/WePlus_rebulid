## 第一章：后台管理系统概述

WePlus 后台管理系统是一个面向开发者和管理员的功能模块，提供用户管理、知识库管理、系统监控等核心功能。系统基于 React + TypeScript 前端和 FastAPI 后端构建，采用 RESTful API 进行数据交互。

### 系统目标
- 提供完整的用户管理和权限控制系统
- 实现知识库文档的增删改查管理
- 提供系统运行状态监控和日志审计
- 支持数据备份和恢复功能

## 第三章：详细开发日志

### 3.1 初始阶段

**目标**: 搭建后台管理系统基础框架

**完成内容**:
1. 创建管理员路由模块 (`admin_routes.py`)
2. 实现用户管理基础 API
3. 搭建前端管理页面框架

**技术决策**:
- 使用 React Router 实现管理页面路由
- 采用 Context API 管理全局状态
- API 请求统一使用 axios 实例

### 3.2 用户管理开发

**目标**: 实现完整的用户管理功能

**完成内容**:
1. 用户列表 API（支持分页、搜索、排序）
2. 用户详情 API
3. 用户创建和编辑 API
4. 用户删除 API
5. 用户状态管理 API
6. 前端用户管理页面（列表、表单、详情）

**遇到的问题**:
- 删除用户时需要清理关联数据（文档、评论等）
- 解决方案：实现级联删除逻辑，先清理关联数据再删除用户

### 3.3 知识库管理开发

**目标**: 实现文档和知识库管理功能

**完成内容**:
1. 文档列表 API（支持分页、搜索、类型筛选）
2. 文档上传 API
3. 文档删除 API
4. 文档分类管理 API
5. 向量索引管理 API
6. 前端知识库管理页面

**遇到的问题**:
- 文档删除时向量数据库不同步问题
- 解决方案：在删除文档时同步调用向量数据库清理接口

### 3.4 系统监控开发

**目标**: 实现系统运行状态监控

**完成内容**:
1. 系统仪表板 API（统计数据聚合）
2. 向量数据库状态监控 API
3. 系统健康检查 API
4. 操作日志 API
5. 前端仪表板页面

**技术实现**:
- 使用聚合查询计算统计数据
- 定时刷新实现实时监控
- 日志记录使用 AOP 思想实现

### 3.5 数据管理开发

**目标**: 实现数据备份和恢复功能

**完成内容**:
1. 数据备份 API（导出 JSON 格式）
2. 数据恢复 API（导入 JSON 格式）
3. 存储统计 API
4. 前端数据管理页面

**安全考虑**:
- 备份文件包含敏感数据，需要访问权限控制
- 恢复操作需要管理员权限
- 备份文件建议加密存储

### 3.6 整合与测试

**目标**: 整合所有模块并进行系统测试

**完成内容**:
1. API 路由整合
2. 前端路由和导航整合
3. 跨模块数据一致性测试
4. 权限控制测试
5. 异常处理测试

## 第五章：技术实现细节

### 5.1 后端架构

```
admin_routes.py
├── GET /dashboard - 获取仪表板数据
├── GET /users - 用户列表（分页）
├── GET /users/{id} - 用户详情
├── PUT /users/{id} - 更新用户
├── DELETE /users/{id} - 删除用户
├── PUT /users/{id}/status - 更新用户状态
├── GET /documents - 文档列表（分页）
├── POST /documents/upload - 上传文档
├── DELETE /documents/{id} - 删除文档
├── GET /vector-index/status - 向量索引状态
├── POST /vector-index/rebuild - 重建向量索引
├── GET /logs - 操作日志
├── POST /backup - 创建备份
├── POST /backup/restore - 恢复备份
└── GET /storage/stats - 存储统计
```

### 5.2 前端状态管理

使用 React Context 管理以下状态：
- **AuthContext**: 用户认证状态和权限信息
- **AdminContext**: 管理页面全局状态（当前选中的用户、筛选条件等）

### 5.3 安全机制

- 所有管理 API 需要 JWT 认证
- 操作需要管理员角色权限
- 敏感操作记录操作日志
- API 访问频率限制

## 第六章：部署注意事项

1. **环境变量配置**
   - 确保 `ADMIN_TOKEN` 或 JWT 密钥已正确配置
   - 设置允许的管理员邮箱白名单

2. **数据库准备**
   - 确保数据库已正确迁移
   - 验证管理员账户已创建

3. **文件存储**
   - 确保上传目录有足够的存储空间
   - 配置备份文件的存储路径

4. **安全配置**
   - 生产环境务必启用 HTTPS
   - 配置 CORS 允许的域名白名单
   - 设置合理的请求频率限制
