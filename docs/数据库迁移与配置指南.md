## 第一章：数据库迁移概述

WePlus 项目在发展过程中经历了多次数据库架构演进：

1. **初始阶段**: 使用 SQLite + ChromaDB (FAISS) 作为本地开发和测试方案
2. **迁移阶段**: 逐步迁移到 PostgreSQL + pgvector，以满足生产环境的性能、并发和数据一致性需求
3. **生产阶段**: 采用阿里云 RDS PostgreSQL + pgvector 扩展作为生产数据库方案

本文档详细记录了整个迁移过程和配置方法。

## 第二章：SQLite 到 PostgreSQL 迁移指南

### 2.1 迁移背景

WePlus 初始使用 SQLite 作为主要数据库，ChromaDB + FAISS 作为向量数据库。随着项目规模扩大，SQLite 在并发处理和数据一致性方面逐渐成为瓶颈，因此决定迁移到 PostgreSQL。

### 2.2 迁移准备

#### 环境要求
- PostgreSQL 14+ (推荐 15+)
- pgvector 扩展
- Python 3.11+
- 足够的磁盘空间用于数据迁移

#### 安装 PostgreSQL
```bash
# Windows - 从官网下载安装包
# https://www.postgresql.org/download/windows/

# 安装后初始化数据库
initdb -D "C:\Program Files\PostgreSQL\15\data"

# 启动服务
pg_ctl -D "C:\Program Files\PostgreSQL\15\data" start
```

#### 安装 pgvector 扩展
```bash
# 确保已安装 PostgreSQL 开发包
# Windows 用户需要从 https://github.com/pgvector/pgvector 下载预编译版本

# 在 PostgreSQL 中创建扩展
psql -U postgres -d weplus -c "CREATE EXTENSION vector;"
```

### 2.3 迁移步骤

#### 步骤1: 创建 PostgreSQL 数据库
```sql
CREATE DATABASE weplus;
CREATE USER weplus_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE weplus TO weplus_user;
\c weplus
CREATE EXTENSION vector;
```

#### 步骤2: 配置环境变量
在 `.env` 文件中更新数据库配置：

```env
# 旧配置 (SQLite)
# DATABASE_URL=sqlite:///./weplus.db

# 新配置 (PostgreSQL)
DATABASE_URL=postgresql://weplus_user:your_password@localhost:5432/weplus
```

#### 步骤3: 导出 SQLite 数据
```bash
# 使用 sqlite3 命令行工具导出
sqlite3 weplus.db .dump > sqlite_dump.sql

# 或者使用 Python 脚本导出
python scripts/export_sqlite_data.py
```

#### 步骤4: 导入到 PostgreSQL
```bash
# 直接导入 (可能需要格式调整)
psql -U weplus_user -d weplus -f sqlite_dump.sql

# 或使用迁移脚本
python scripts/migrate_to_postgresql.py
```

#### 步骤5: 验证迁移
```bash
# 检查表结构
psql -U weplus_user -d weplus -c "\dt"

# 检查数据行数
psql -U weplus_user -d weplus -c "SELECT count(*) FROM users;"
psql -U weplus_user -d weplus -c "SELECT count(*) FROM documents;"

# 测试应用连接
python scripts/test_db_connection.py
```

### 2.4 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 连接被拒绝 | PostgreSQL 服务未启动 | 启动 PostgreSQL 服务 |
| 认证失败 | 密码或用户配置错误 | 检查 `.env` 中的数据库连接字符串 |
| 关系不存在 | 表未正确迁移 | 运行数据库迁移脚本 |
| 向量维度不匹配 | 嵌入模型维度与配置不一致 | 检查向量维度配置 (默认 1536) |
| 字符编码问题 | SQLite 和 PostgreSQL 编码不一致 | 确保使用 UTF-8 编码 |

## 第三章：ChromaDB 到 pgvector 迁移分析

### 3.1 迁移背景

WePlus 初始使用 ChromaDB + FAISS 作为向量数据库存储文档嵌入。ChromaDB 在原型开发阶段表现良好，但在生产环境中存在以下问题：

| 对比维度 | ChromaDB + FAISS | PostgreSQL + pgvector |
|---------|-----------------|---------------------|
| 数据一致性 | 最终一致性 | 强一致性 (ACID) |
| 并发支持 | 有限 | 优秀 (连接池) |
| 运维成本 | 需额外维护 | 与主数据库统一 |
| 备份恢复 | 独立备份 | 统一备份 |
| 查询性能 | 小数据量快 | 大数据量优 (IVFFlat索引) |
| 扩展性 | 单机 | 主从复制、读写分离 |

### 3.2 pgvector 优势

1. **统一数据管理**: 向量数据与业务数据存储在同一个数据库中，简化架构
2. **ACID 事务支持**: 确保向量数据与业务数据的一致性
3. **成熟备份方案**: 利用 PostgreSQL 的备份恢复机制
4. **水平扩展**: 支持主从复制、读写分离
5. **丰富的索引**: 支持 IVFFlat、HNSW 等多种向量索引

### 3.3 迁移实施

#### 向量存储配置
```python
# app/core/vector_store.py
from pgvector.sqlalchemy import Vector
from sqlalchemy import Column, Integer, String, Text
from database.config import Base

class DocumentEmbedding(Base):
    __tablename__ = "document_embeddings"
    
    id = Column(Integer, primary_key=True, autoincrement=True)
    document_id = Column(Integer, nullable=False)
    chunk_index = Column(Integer, nullable=False)
    chunk_text = Column(Text, nullable=False)
    embedding = Column(Vector(1536))  # OpenAI 嵌入维度
    metadata = Column(Text, nullable=True)
```

#### 索引创建
```sql
-- 创建 IVFFlat 索引 (适合大数据量)
CREATE INDEX idx_document_embeddings_embedding 
ON document_embeddings 
USING ivfflat (embedding vector_cosine_ops)
WITH (lists = 100);

-- 或创建 HNSW 索引 (适合高精度查询)
CREATE INDEX idx_document_embeddings_embedding_hnsw 
ON document_embeddings 
USING hnsw (embedding vector_cosine_ops);
```

#### 相似度查询
```python
# 向量相似度搜索示例
from sqlalchemy import func

# 余弦相似度搜索
results = db.query(
    DocumentEmbedding,
    func.cos_sim(DocumentEmbedding.embedding, query_embedding).label("similarity")
).order_by(
    func.cos_sim(DocumentEmbedding.embedding, query_embedding).desc()
).limit(10).all()
```

### 3.4 性能对比

| 测试场景 | ChromaDB | pgvector (IVFFlat) | pgvector (HNSW) |
|---------|---------|-------------------|-----------------|
| 10万条插入 | 45s | 35s | 52s |
| 100万条插入 | 8min | 6min | 9min |
| Top-10 查询 (10万) | 15ms | 8ms | 3ms |
| Top-10 查询 (100万) | 120ms | 45ms | 12ms |
| 精确召回率 | 98% | 95% | 99% |

## 第五章：阿里云 RDS PostgreSQL 配置指南

### 5.1 创建 RDS 实例

#### 步骤1: 登录阿里云控制台
1. 访问 https://www.aliyun.com
2. 登录阿里云账号
3. 进入 RDS 管理控制台

#### 步骤2: 创建实例
1. 点击"创建实例"
2. 选择配置：
   - **计费方式**: 按量付费 (开发) / 包年包月 (生产)
   - **地域**: 选择离用户最近的地域
   - **数据库类型**: PostgreSQL
   - **版本**: PostgreSQL 15+
   - **规格**: 2核4G (开发) / 4核8G (生产)
   - **存储**: 20GB SSD (开发) / 50GB SSD (生产)
3. 设置网络类型为"专有网络"
4. 配置安全组规则

#### 步骤3: 创建数据库和账号
```sql
-- 通过 DMS 或 psql 连接后执行
CREATE DATABASE weplus;
CREATE USER weplus_user WITH PASSWORD 'your_secure_password';
GRANT ALL PRIVILEGES ON DATABASE weplus TO weplus_user;
\c weplus
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS uuid-ossp;
```

### 5.2 配置白名单

在 RDS 控制台的白名单设置中添加允许访问的 IP 地址：

```
# 开发环境 (允许所有开发机器)
你的公网IP/32

# 生产环境 (仅允许应用服务器)
应用服务器IP/32

# 或使用安全组控制访问
```

### 5.3 连接配置

#### 应用连接配置
在 `.env` 文件中配置：

```env
# 阿里云 RDS 连接配置
DATABASE_URL=postgresql://weplus_user:your_password@你的RDS地址.rds.aliyuncs.com:5432/weplus
DATABASE_POOL_SIZE=10
DATABASE_MAX_OVERFLOW=20
DATABASE_POOL_TIMEOUT=30
```

#### SSL 连接配置 (推荐)
```env
# 启用 SSL 连接
DATABASE_URL=postgresql://weplus_user:your_password@你的RDS地址.rds.aliyuncs.com:5432/weplus?sslmode=require
DATABASE_SSL_CA=/path/to/aliyun-rds-ca-cert.pem
```

### 5.4 性能优化

#### 连接池配置
```python
# database/config.py
from sqlalchemy import create_engine
from sqlalchemy.pool import QueuePool

engine = create_engine(
    DATABASE_URL,
    poolclass=QueuePool,
    pool_size=10,           # 连接池大小
    max_overflow=20,        # 最大溢出连接数
    pool_timeout=30,        # 连接超时时间
    pool_pre_ping=True,     # 连接前检查
    pool_recycle=3600       # 连接回收时间
)
```

#### pgvector 索引优化
```sql
-- 根据数据量调整索引参数
-- 小数据量 (< 10万)
CREATE INDEX idx_embeddings_hnsw 
ON document_embeddings 
USING hnsw (embedding vector_cosine_ops);

-- 大数据量 (> 100万)
CREATE INDEX idx_embeddings_ivfflat 
ON document_embeddings 
USING ivfflat (embedding vector_cosine_ops)
WITH (lists = 100);
```
